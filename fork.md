# Gemma 4 Fine-Tuned Model → Browser ONNX: Status & Next Steps

## What We Faced

We successfully fine-tuned Google's Gemma 4 E2B model on IRS tax documentation using Unsloth Studio on Colab, producing a merged 16-bit model (`madhured/irs-gemma4-e2b-merged-unsloth`). The challenge came at the last mile: converting this PyTorch model into ONNX format that runs in the browser via WebGPU. We attempted every available path — `optimum-cli export onnx` (unsupported architecture error), `ORTModelForCausalLM.from_pretrained(export=True)` (same error), `torch.onnx.export` with legacy TorchScript (masking_utils crash), `torch.jit.trace` (same crash), and `torch.onnx.export` with dynamo (successfully creates `model.onnx` but ONNX Runtime Web cannot load it due to Gemma 4's Per-Layer Embeddings, variable attention head dimensions, and shared KV cache — operations the browser runtime doesn't support yet). The root cause is confirmed by open issues on both `huggingface/optimum-onnx` (#49, #67) and `microsoft/onnxruntime-genai` (#2062): Gemma 4 (and even Gemma 3) ONNX export is not yet supported in any public toolchain. The only working browser ONNX model (`onnx-community/gemma-4-E2B-it-ONNX`) was created by HuggingFace staff using custom internal scripts not available publicly.

## Path Forward

The long-term solution is either forking `huggingface/optimum-onnx` and adding a `Gemma4OnnxConfig` class (modeled after the existing `gemma3_text` config, with additions for PLE, variable head dims, and shared KV cache), or waiting for the optimum-onnx team to add native support. The fork approach is roughly a day of focused work and would be a valuable open-source contribution — but even a successful export may still hit ONNX Runtime Web limitations in the browser. For immediate deployment, the model works today via Ollama (GGUF export from Unsloth Studio) served locally with the browser calling its API, or by using the base `onnx-community/gemma-4-E2B-it-ONNX` model in the browser with IRS expertise injected through the system prompt.
